# 进入逻辑与密码逻辑说明

## 1. 系统定位

本系统分为两层：

- 门户层：负责固定入口、统一口令、分组入口、总控入口、二维码分发
- 会议层：由 `BigBlueButton` 真正承载音视频会议

也就是说，用户先通过门户进入，门户再自动跳转到对应的 `BigBlueButton` 房间。

## 2. 四类入口

### 分会场入口

- 路径：`/join/<room-slug>`
- 适用对象：每个分会场终端
- 权限：普通参会

### 房间主持入口

- 路径：`/moderator/<room-slug>`
- 适用对象：负责该分会场的主持人
- 权限：该房间主持人

### 分组主持入口

- 路径：`/group/<group-slug>`
- 适用对象：分组主持人
- 权限：先进入分组工作台，再选择本组分会场的主持入口

### 总主持人入口

- 路径：`/master`
- 适用对象：总主持人
- 权限：查看全部分组和全部分会场

## 3. 三层密码

### 每日统一口令

- 这是每天会前人工通知给所有人的统一口令
- 分会场、分组主持、总主持人进入门户时都要填写
- 在门户管理后台修改

### BigBlueButton 房间内部密码

- 系统为每个房间保存一组 `attendeePassword` 和 `moderatorPassword`
- 这些密码不需要人工分发给终端
- 门户会在后台自动使用它们生成真实入会地址

### 后台管理员密码

- 这是专门进入 `/admin` 管理页的密码
- 它不等于每日统一口令

## 4. 进入流程

1. 管理员进入后台修改“每日统一口令”。
2. 将固定链接或二维码分发给对应分会场和主持人。
3. 用户打开自己的入口地址。
4. 用户输入姓名和每日统一口令。
5. 门户自动调用 `BigBlueButton create/join`，跳转入会。

## 5. 轮巡查看逻辑

- 分组主持人通过 `/group/<group-slug>` 进入本组工作台
- 从工作台批量打开本组多个主持标签页
- 在 `Edge` 中配合 `Revolver Tabs` 做自动轮巡

## 7. 推荐管理动作

- 每天会前只改“每日统一口令”
- 房间 slug、meetingId、分组关系尽量长期固定
- 后台管理员密码不要频繁对外分发，仅限少数运维人员掌握
